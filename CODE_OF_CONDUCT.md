# Code of Conduct

## The Short Version

Be direct. Be honest. Be respectful.
Treat everyone the way you want to be treated.
Focus on the work, not on the person.

---

## The Longer Version

Korers is a small project built in public. The community around it should reflect the same values the product is built on: transparency, honesty, and no unnecessary complexity.

**Expected behavior:**

- Communicate clearly and directly
- Give feedback on work, not on people
- Accept that others may have different approaches and explain yours instead of dismissing theirs
- Acknowledge when you are wrong
- Ask questions when something is unclear instead of assuming

**Not acceptable:**

- Personal attacks or insults
- Harassment of any kind
- Dismissing contributions without explanation
- Spam or self-promotion unrelated to the project
- Sharing private information about others without their consent

---

## Scope

This Code of Conduct applies to all project spaces including GitHub Issues, Discussions, Pull Requests, and any other communication related to this project.

---

## Enforcement

This project is currently maintained by one person.

If something happens that violates this Code of Conduct, contact the maintainer directly via GitHub.

Reports will be handled as quickly as possible.
If the behavior continues after being addressed, the person will be removed from the project spaces.

---

## Philosophy

Security tools attract people who think carefully about trust and systems.
This community should work the same way.
No theatrics, no politics, just people working on something that matters.
